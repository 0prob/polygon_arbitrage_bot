createEffect has a catch...

createEffect is part of the HyperIndex framework runtime — it requires config.yaml, indexer.onEvent handlers, and the full framework lifecycle. You can't import it standalone.

So "createEffect Only" really means one of these:

A

Build our own Effect system

Replace mapWithConcurrency + LRU cache + retry with a typed, batched, auto-dedup RPC effect layer. Same pattern as createEffect but homegrown. No framework migration.

B

Minimal HyperIndex setup (effects + SQLite only)

Set up a minimal HyperIndex project that only handles RPC effects and writes pool state to a shared SQLite DB. The bot's pass loop reads from the DB. No event handlers.

C

Just Bun + keep current architecture

Switch to Bun runtime for faster startup. Replace mapWithConcurrency with Bun's native Promise.allSettled + concurrency patterns. Keep everything else as-is.

Which sounds right? Or is there something specific about createEffect you want?
